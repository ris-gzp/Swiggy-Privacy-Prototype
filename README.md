# Swiggy Privacy Prototype — DPDP Act Compliance

🔗 **Live Demo:** https://ris-gzp.github.io/swiggy-privacy-prototype/

## Overview

A clickable, end-to-end prototype demonstrating how Swiggy could redesign its 
consent and privacy experience to comply with India's Digital Personal Data 
Protection (DPDP) Act. This was built as part of a Product Thinking exercise, 
covering the complete happy-path user journey from consent collection to 
data-rights management.

## Problem Statement

With the DPDP Act mandating explicit, granular, and revocable user consent for 
personal data processing, platforms like Swiggy need a consent architecture 
that is both legally compliant and doesn't hurt conversion or user experience. 
This prototype explores that balance.

## What's Inside

- **Consent Collection Flow** — granular, purpose-specific consent at signup/first use
- **Privacy Dashboard** — a single place for users to view and manage what data is being used and why
- **Consent Withdrawal** — clear, low-friction path to revoke consent per data category
- **Data Rights Center** — access, correction, and erasure requests handled transparently

## Product Thinking

This prototype is paired with a full Product Thinking Deck covering:
- User empathy and friction points in existing consent flows
- Regulatory requirements mapped to product decisions
- Success metrics (consent completion rate, revocation friction, support ticket deflection)
- Trade-offs between compliance rigor and conversion impact

## Tech

Built as a static HTML/CSS/JS clickable prototype — happy path only, no backend, 
designed for quick review and walkthrough.
